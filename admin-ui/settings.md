# Settings

The project settings are available under the **Settings** page which is accessible by clicking on the Settings menu item on the left sidebar.

On this page you can find various sections available in Tabs:

- General Settings
- Admin Accounts
- Admin Roles
- Api Tokens
- Webhooks
- Billing

## General

**Project:** Edit the Display Name, description and the Instance name (project name) used to access the API.

**Endpoints:** Visualize and copy the instance name (project name), the GraphQL API Endpoint and the REST API endpoint.

**Internazionalization:** Manage the active languages available for the project.

## Members

List of all members who can access the project.

Invite new members via email and assign Roles for the project.

## Roles

List of all roles that can be used in the project.

Assign granular permissions to each of them.

## API Tokens

List of all API Tokens you can use to access the project.

Assign to each API token granular permissions and the range of dates (optional) it is enabled.

For more information about authentication, see [API Basics](/api-overview/api-basics#authentication-authorization).

## Webhooks

List of all webhooks created for the project.

Configure how the webhooks should send data, if they are active, authentication, headers etc.

## Billing

- Check the quota usage based on your plan
- Download invoices
- Change subscription plan
